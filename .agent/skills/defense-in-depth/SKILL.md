# 多層防御スキル (Defense in Depth)

コード変更時にセキュリティを多層的に確認し、金融データを保護する。

> ⚠️ `money-master` と `quant-brain` は金融データ/取引を扱うため、特に注意が必要。

---

## いつ使うか

- API エンドポイントを追加/変更する時
- 認証/認可に関わるコードを変更する時
- データベーススキーマを変更する時
- 外部API連携を追加する時
- 環境変数やシークレットを扱う時

---

## セキュリティチェック階層

### Layer 1: 入力バリデーション

```typescript
// ✅ Zod でサーバーサイドバリデーション
import { z } from 'zod';

const schema = z.object({
  amount: z.number().positive(),
  ticker: z.string().min(1).max(10),
  date: z.string().regex(/^\d{4}-\d{2}-\d{2}$/),
});
```

| チェック項目 | 確認 |
|:---|:---|
| すべてのAPIエンドポイントに入力バリデーションがあるか | |
| SQLインジェクション対策 (パラメータバインド) | |
| XSS対策 (ユーザー入力のエスケープ) | |

### Layer 2: 認証・認可

| チェック項目 | 確認 |
|:---|:---|
| 管理APIに環境チェックがあるか (`NODE_ENV !== 'production'`) | |
| デバッグAPIが本番で無効化されているか | |
| CORS設定が適切か | |

### Layer 3: データ保護

| チェック項目 | 確認 |
|:---|:---|
| APIキーがコードにハードコードされていないか | |
| `.env` ファイルが `.gitignore` に含まれているか | |
| エラーレスポンスに内部情報が漏洩していないか | |
| DBバックアップが設定されているか | |

### Layer 4: 依存関係

// turbo
```powershell
# 既知の脆弱性チェック
pnpm audit

# Python 依存関係
docker exec <python-container> pip audit 2>/dev/null || echo "pip-audit not installed"
```

### Layer 5: Docker/ネットワーク

| チェック項目 | 確認 |
|:---|:---|
| 不要なポートが公開されていないか | |
| コンテナが root で実行されていないか | |
| ボリュームマウントが必要最小限か | |

## 金融アプリ固有のチェック

### money-master (ポート 3001)
- [ ] 資産データの改ざん防止
- [ ] 取引記録の整合性
- [ ] バックアップの定期実行

### quant-brain (ポート 8002)
- [ ] APIキー (取引所) の安全な管理
- [ ] 注文パラメータのバリデーション
- [ ] ゴーストクライアント ↔ 実取引の明確な分離

---

## チェックリスト (最低限)

- [ ] 入力バリデーション確認
- [ ] 管理API/デバッグAPIのアクセス制御確認
- [ ] シークレット/APIキーがコードに含まれていないか確認
- [ ] `pnpm audit` で脆弱性チェック
- [ ] エラーレスポンスに内部情報が漏洩していないか確認

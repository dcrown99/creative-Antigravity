# UI変更検証スキル

UI（画面）に影響を与える変更を行った場合、**必ずブラウザで動作確認**してから完了報告を行う。

---

## 適用条件

以下のいずれかに該当する場合、このスキルを適用:

1. **コンポーネント変更**: `.tsx` ファイルの編集
2. **データ変更**: DBの INSERT/UPDATE/DELETE でUIに表示されるデータを変更
3. **API変更**: フロントエンドが使用するAPIエンドポイントの変更
4. **スタイル変更**: CSS/Tailwind の変更

---

## 検証手順

### Step 1: 対象URLの特定
変更が反映される画面のURLを特定する。

```
| アプリ | ポート | 例 |
|:---|:---|:---|
| money-master | 3001 | http://localhost:3001/assets |
| my-kindle | 3002 | http://localhost:3002 |
| auto-clipper-web | 3003 | http://localhost:3003 |
| ai-talker | 3004 | http://localhost:3004 |
| news-reader | 3005 | http://localhost:3005 |
```

### Step 2: ブラウザで確認
// turbo
```
browser_subagent を使用して以下を実行:
1. 対象URLを開く
2. 変更箇所が正しく表示されているか確認
3. 必要に応じて操作（クリック、入力など）を実行
4. スクリーンショットを取得
```

**重要**: `browser_subagent` が失敗した場合でも、ユーザーに確認URLを提示して手動確認を依頼する。

### Step 3: 検証結果の記録
検証チケットに以下を記載:

```markdown
## ブラウザ検証
- URL: http://localhost:XXXX/path
- 確認項目: [変更内容]
- 結果: ✓ 正常 / ✗ 問題あり
- スクリーンショット: (あれば埋め込み)
```

### Step 4: 問題発見時
問題を発見した場合:
1. 原因を特定
2. 修正を実施
3. **再度 Step 2 から実行**
4. 問題が解消されるまで繰り返す

---

## チェックリスト

完了報告前に確認:

- [ ] 変更箇所がブラウザで正しく表示されている
- [ ] 関連する他の画面に悪影響がない
- [ ] 検証チケットにブラウザ検証結果を記載した

---

## 注意事項

1. **DB変更だけでは不十分**: SQLクエリで正しい値を確認しても、UIで正しく表示されるとは限らない
2. **API確認だけでは不十分**: APIレスポンスが正しくても、フロントエンドの表示ロジックに問題がある可能性がある
3. **ユーザーに丸投げしない**: 「ブラウザで確認してください」で終わらせず、AIが先に確認する

# 振り返りと課題：配当履歴復旧プロジェクト (2025-12-26)

## 1. 実施内容
失われていた配当履歴データの復旧および、表示データの適正化（日本円統一）を行いました。

- **データ復旧:** Yahoo Finance API (`yf.chart`) を使用して過去の配当履歴を取得。
- **データ適正化:**
  - `1株あたり配当金` → `総受取額` (×現在の保有数) に変換。
  - `USD` → `JPY` (×過去の為替レート) に変換。
- **UI表示:** `Money Master` 配当一覧画面での表示確認。

## 2. 発生した問題と原因
### (1) データベースファイルの不整合
**事象:** スクリプトで復旧したはずのデータが画面に表示されない。
**原因:**
- **スクリプト実行環境:** `apps/money-master/.env` を参照し、`apps/money-master/data/money-master.db` に書き込んでいた。
- **Docker実行環境:** `docker-compose.dev.yml` で `root/data/money-master.db` をマウントしていた。
- 結果として、**「開発用DBに書き込み、本番用DBを表示する」** 状態となり、データが表示されなかった。
**解決:** 修復スクリプト実行時に `DATABASE_URL` 環境変数をルート上のDBファイルへの絶対パス（`file:C:/.../code/data/money-master.db`）で上書き指定することで解決。

### (2) 日付データの破損
**事象:** 復旧初期、日付が「未来の日付（+57745年など）」で登録された。
**原因:** Yahoo Finance APIが返すタイムスタンプ（秒・ミリ秒）の判定ロジックが不十分だった。
**解決:** タイムスタンプの桁数判定と`Date`オブジェクト変換ロジックを強化し、異常値を除去（`cleanup_bad_data.ts`）。

## 3. 今後の課題 (Backlog ideas)

### 🔹 自動化の強化
現在は手動スクリプト (`recover-dividends`) で実行しているが、定期的に（例：月次）自動実行し、新しい配当落ち情報を自動登録する仕組みが必要。
- **案:** `market-watcher` または `auto-clipper-worker` に定期ジョブとして組み込む。

### 🔹 データベースパスの統一 (SSOT)
`.env` ファイルによって参照先DBが分散するリスクがある。
- **案:** Docker経由での実行 (`docker compose exec money-master ...`) を標準とし、ホスト側からの直接書き込みを原則禁止にするか、`.env` のパスをルートDBへの絶対パス/相対パスで厳密に固定する。

### 🔹 為替レート精度の向上
現在は「配当支払い当日」の終値を使用しているが、実際の為替（入金時のレート）とは多少の誤差がある。
- **案:** 証券会社のCSVインポート機能を作成し、正確な受取額を取り込めるようにする。

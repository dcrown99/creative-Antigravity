# Python Environment Migration to `uv`

## 概要 (Overview)
プロジェクト全体のPython環境管理をレガシーな `Poetry` や `pip` から、高速かつ統合された Rust 製パッケージマネージャ **`uv`** へ完全移行しました。
これにより、コンテナビルドの大幅な高速化と、各ツール・スクリプト間での一貫した実行環境の提供が実現されました。

## 変更内容 (Changes Made)

### 1. アプリケーションの移行
以下のPythonベースのアプリケーションにおいて、パッケージ管理手法を統一しました。
- `quant-brain`: Poetry を廃止し、PEP 621 準拠の `[project]` テーブルに移行。
- `market-watcher`: `requirements.txt` を廃止し、`pyproject.toml` に移行。
- `auto-clipper-api`: 複数に分かれていた `requirements.txt` / `-dev.txt` を統合。
- `manga-downloader`: 専用の `pyproject.toml` を作成し、バッチスクリプトを更新。

**全アプリケーション共通:**
- Dockerfile のビルドステップを `uv sync --frozen` に置き換え、超高速かつ確実なイメージビルドを実現。
- ローカル環境でのセットアップ用スクリプト (`setup_env.ps1`, `setup_env.sh`, `漫画ダウンローダー.bat`) において作成処理を `uv run` または `uv sync` に切り替え。

### 2. 管理スクリプトの更新
システム全体に影響を及ぼす以下の管理スクリプトを `uv` 対応させました。
- `repair_dependencies.ps1`: Node.js に加え、Python コンテナに対しても `uv sync` によるホットリロードをサポート。
- `lint_python.ps1`: 以前はローカルコンテキストに依存していたRuffの実行を、`uv run ruff check .` に統一して全コンテナおよびローカルツールに適用。

### 3. ドキュメントの最新化
以下のドキュメントにおいて、古い `pip` や `poetry`、仮想環境 (`python -m venv`) の記述を最新の `uv` コマンドラインにアップデートしました。
- 全体の `ADR.md` に今回の戦略決定を **ADR-023: Python環境管理の `uv` への完全移行** として追加。
- 各アプリケーションごとの `README.md` および `使い方.md`、`exe化手順.md` などユーザー向けドキュメント。
